1. System Overview

The Bank Management System is a desktop application developed in Java with a graphical user interface built using Java Swing. It simulates basic banking operations, providing an interactive platform for managing customer accounts, transactions, and other banking services.

Technology Stack

  • Programming Language: Java 8+
  • GUI Framework: Java Swing
  • Database: MySQL 8.0+
  • Database Connectivity: JDBC (Java Database Connectivity)
  • Architecture Pattern: MVC (Model-View-Controller) with Swing components

Key Features

  • Login System: Secure authentication for bank customers
  • Account Management: Create, view, edit, and manage customer accounts
  • Transaction Handling: Facilitate deposits, withdrawals, and balance inquiries
  • Balance Inquiry: Check account balances in real-time
  • Change PIN: Secure PIN modification functionality
  • E-statement: View transaction history and account statements
  • Fast Cash: Quick withdrawal with predefined amounts

System Requirements

  • Java Development Kit (JDK) 8 or above
  • MySQL Server 8.0+
  • MySQL Connector/J (JDBC driver)
  • Minimum 4GB RAM
  • Windows/Linux/macOS operating system

3. Module-by-Module Validation Details

1. Login Module (Login.java)

Purpose: Authenticate users with card number and PIN credentials.

Form Fields and Validation Rules:

Card Number Field (textField2)

  • Type: JTextField
  • Validation: Required field, must match existing card number in database
  • Business Logic: Card number must exist in login table

PIN Field (passwordField3)

  • Type: JPasswordField
  • Validation: Required field, must match PIN for the given card number
  • Business Logic: PIN must match the stored PIN for the card number

2. Deposit Module (Deposit.java)

Purpose: Process cash deposits into customer accounts.

Amount Field (textField)

  • Type: TextField
  • Validation: Required field, cannot be empty
  • Business Logic: Must be a positive numeric value

3. Withdrawal Module (Withdrawl.java)

Purpose: Process cash withdrawals from customer accounts.

Amount Field (textField)

  • Type: TextField
  • Validation: Required field, cannot be empty, must not exceed balance
  • Business Logic: Maximum withdrawal limit is Rs. 10,000

4. Database Schema & Relationships

Database Overview

The system uses a MySQL database named bankSystem with the following tables:

  • login: Authentication credentials
  • signup: Personal information (Page 1)
  • signuptwo: Additional details (Page 2)
  • signupthree: Account details (Page 3)
  • bank: Transaction records
