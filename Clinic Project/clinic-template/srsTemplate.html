1. Introduction

1.1 Document Purpose

This document specifies the functional and non-functional requirements for the Clinic Management System (CMS) â€” a full-stack, web-based clinic management application implemented in the public GitHub repository kongjiyu/ClinicManagementSystem. The SRS covers the entire application as packaged in the repository (backend services, frontend JSP-based UI, database initialization, and dockerized deployment). The SRS is intended to support implementation verification, acceptance testing, and future maintenance. Repo evidence: README (project overview and feature list). GitHub

1.2 Product Scope

CMS is a full-stack web application that streamlines clinic operations: patient and doctor management, appointment scheduling, medicine inventory and prescriptions, billing/payments, and analytics and reporting (patient trends, doctor performance, medicine sales, consultation analysis). It is intended for small-to-medium clinics that want a self-contained web app deployable via Docker or in an application server such as GlassFish. The system provides a responsive UI and exportable reports. GitHub
