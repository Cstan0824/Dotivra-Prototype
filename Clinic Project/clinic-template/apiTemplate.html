Clinic Management System API Documentation

Overview

This document provides comprehensive API documentation for the Clinic Management System, including all endpoints, request/response formats, and error handling.

Base URL

http://localhost:8080/ClinicManagementSystem/api

Authentication

Most endpoints require authentication. Use the /auth/login endpoint to obtain a session.

Common Response Format

All API responses follow this structure:

{ "success": true/false, "message": "Response message", "data": {...} }

Error Response Format

{ "error": "Error message description", "timestamp": "2024-01-01T00:00:00" }

1. Authentication Module (/auth)

1.1 User Login

POST /auth/login

Description: Authenticate staff or patient users

Request Body:

{ "username": "string", "password": "string", "userType": "staff|patient", "rememberMe": boolean }

Response:

{ "success": true, "userType": "staff|patient", "message": "Login successful", "userName": "string", "userRole": "string" // Only for staff }

Error Codes: 400, 500

1.2 User Logout

POST /auth/logout

Description: End user session

Response:

{ "success": true, "message": "Logout successful" }

Error Codes: 500

1.3 Get Session Status

GET /auth/session

Description: Check current authentication status

Response:

{ "authenticated": true, "userType": "staff|patient", "userId": "string", "userName": "string", "userRole": "string" // Only for staff }

Error Codes: 500

2. Patient Management Module (/patients)

2.1 Get All Patients

GET /patients

Description: Retrieve all patients sorted by name

Response: Array of Patient objects

Error Codes: 500

2.2 Get Patient by ID

GET /patients/{id}

Description: Retrieve specific patient details

Path Parameters: id - Patient ID

Response: Patient object

Error Codes: 404, 500

2.3 Create Patient

POST /patients

Description: Register new patient

Request Body: Patient object

Response: Created Patient object (201)

Error Codes: 400, 500

3. Appointment Management Module (/appointments)

3.1 Create Appointment

POST /appointments

Description: Schedule new appointment

Request Body: Appointment object

Response: Created Appointment object (201)

Error Codes: 400, 409 (conflict - existing appointment)

3.2 Get All Appointments

GET /appointments

Description: Retrieve all appointments sorted by date/time

Response: Array of AppointmentWithPatientDTO objects

Error Codes: 500

3.3 Reschedule Appointment

PUT /appointments/{id}/reschedule

Description: Change appointment time

Path Parameters: id - Appointment ID

Request Body: New date-time string (yyyy-MM-ddTHH:mm:ss)

Response: Updated Appointment object

Error Codes: 400, 500

4. Consultation Management Module (/consultations)

4.1 Get All Consultations

GET /consultations

Description: Retrieve all consultations sorted by date

Response: Array of Consultation objects

Error Codes: 500

4.2 Create Consultation

POST /consultations

Description: Create new consultation

Request Body: Consultation object

Response: Created Consultation object (201)

Error Codes: 400, 500

4.3 Create Medical Certificate (MC)

POST /consultations/{id}/mc

Description: Create MC for consultation

Path Parameters: id - Consultation ID

Request Body:

{ "startDate": "YYYY-MM-DD", "endDate": "YYYY-MM-DD", "description": "string" }

Response: Updated Consultation object (201)

Error Codes: 400, 404, 500

5. Queue Management Module (/queue)

5.1 Get All Queues

GET /queue

Description: Get today's queue data (appointments and consultations)

5.2 Check-in Appointment

POST /queue/checkin-appointment/{appointmentId}

Description: Check-in appointment and create consultation

Path Parameters: appointmentId - Appointment ID

Response: Created Consultation object

Error Codes: 400, 500

9.1 Create Bill from Consultation

POST /bills/from-consultation/{consultationId}

Description: Generate bill from consultation

Path Parameters: consultationId - Consultation ID

GET /bills/{id}/pdf

10. Reporting Module (/reports)

10.1 Patient Registration Report

GET /reports/patient-registration

Description: Get patient registration statistics

Query Parameters:

  • period - daily, weekly, monthly, yearly
  • startDate - Start date (YYYY-MM-DD)
  • endDate - End date (YYYY-MM-DD)

Response: Patient registration statistics

Error Codes: 500

10.2 Consultation Volume Report

GET /reports/consultation-volume

Description: Get consultation volume statistics

Query Parameters:

  • period - daily, weekly, monthly, yearly
  • startDate - Start date (YYYY-MM-DD)
  • endDate - End date (YYYY-MM-DD)

Response: Consultation volume statistics

Error Codes: 500

12. Error Handling

12.1 HTTP Status Codes

  • 200 - OK (Success)
  • 201 - Created (Resource created successfully)
  • 400 - Bad Request (Invalid input data)
  • 404 - Not Found (Resource not found)
  • 409 - Conflict (Resource conflict)
  • 500 - Internal Server Error (Server error)

12.2 Common Error Scenarios

  • Invalid ID format - 400 Bad Request
  • Resource not found - 404 Not Found
  • Duplicate resource - 409 Conflict
  • Validation errors - 400 Bad Request
  • Authentication required - 401 Unauthorized
  • Server errors - 500 Internal Server Error

13. Integration Guidelines

13.1 Authentication Flow

  1. Call /auth/login with credentials
  2. Store session cookie from response
  3. Include session cookie in subsequent requests
  4. Call /auth/logout to end session

13.2 Best Practices

  • Use HTTPS in production
  • Implement proper error handling
  • Cache frequently accessed data
  • Use pagination for large datasets
  • Implement retry logic for failed requests
